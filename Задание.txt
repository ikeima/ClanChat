Цель задания:

Необходимо разработать набор приложений для реализации кланового чата.

Клиентское приложение можно выполнить в виде веб-приложения или консольного приложения на усмотрение кандидата. Клиентская часть должна дать возможность указать название клана и имя игрока для подключения к чату.

Требования:

У каждого клана свой чат (своя комната) для общения. Единовременно клиент должен быть подключен только к одному чату - чату указанного клана;
При подключении клиент получает с сервера последние 50 сообщений. В подключенный клиент сообщения должны приходить с задержкой не более 1 секунды;
Сервис чата должен быть отказоустойчивым. В случае перезапуска сервисов чата история клановых чатов не должна быть потеряна. Выбор БД на усмотрения кандидата.
Дополнительные задания:

Использование авторизации для входа в чат, чтобы игрок не мог читать сообщения в “чужом” клане;
Наличие Unit-тестов.
Оценка задания будет основываться на следующих критериях:

Применение ООП принципов.
Корректная реализация архитектурного паттерна (MVP или MVC) и понимание его принципов.
Качество и чистота кода, включая именование переменных, комментарии и организацию файлов.
Исходный код проекта разместить на GitHub или GitLab.
